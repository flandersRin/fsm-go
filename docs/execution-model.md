# 执行模型

workflow-go 的核心原则是状态和任务分离。

状态只回答：流程现在在哪里。

任务回答：现在要执行什么工作。

## 启动流程

`StartWorkflow` 创建 `WorkflowInstance`，记录启动历史，并调度初始状态的入口任务。

## 发送事件

`SignalWorkflow` 接收外部事件，根据当前状态和事件找到转移规则，推进状态并记录时间线。

## 执行任务

`RunDueTasks` 扫描到期任务，抢占任务，调用注册的任务处理器，然后根据结果推进流程。

任务成功后可以发出事件。任务失败后由运行时决定是否重试、失败或补偿。

## 时间线

每次流程启动、事件进入、状态变化、任务调度、任务执行、重试、失败、补偿都会写入 `ExecutionHistory`。
